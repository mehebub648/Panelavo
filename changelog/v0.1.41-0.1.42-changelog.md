# 0.1.41 to 0.1.42

- Made a private rootless Docker daemon the default runtime for each Docker site user, with no root-daemon fallback.
- Added Super Admin initialization checks and fixes for Docker rootless packages, subordinate IDs, the systemd user manager, storage, cgroups, networking, and disk readiness.
- Allowed site-write users to manage only their own rootless Compose projects while retaining safety scanning and loopback port verification.
- Added resumable preparation, journaled ownership translation, verified cutover, rollback, and recovery for legacy rootful bind-mount projects. Named and external volume migration remains unsupported.
- Added ordered rootless cleanup before a Docker site's Unix user is deleted.
- Bumped the root broker protocol from 2 to 3. Existing installations must run the trusted `setup.sh` before the in-panel updater can install this release.

# 0.1.20

- Generalized preflight click-to-fix beyond host-software installs: any blocked check Panelavo can safely resolve now shows a fix button, and fixes are split into host-scoped (Super Admin) and site-scoped (ordinary Operations permission) categories.
- Added a site-scoped "Bind published ports to localhost" fix for the Docker Compose host-safety policy. When a project is unsafe solely because a published port is not bound to the loopback interface, one click rewrites the Compose file so each published port binds to `127.0.0.1` — the address the site's reverse proxy already targets — keeping the same port numbers.
- The port rewrite only touches entries inside `ports:` blocks (container ports, environment values, and comments are never modified), handles short list syntax and long-syntax `host_ip:` lines, and leaves forms it will not rewrite textually (bracketed IPv6, non-numeric hosts, flow syntax) untouched.
- The edit is staged to a temporary file, validated with `docker compose config`, and re-scanned against the full host-safety policy; it is committed only when the result is fully safe, and the original is backed up as `<file>.panelavo.bak`. The operation output includes the exact line changes.
- The fix is offered only when published-port bindings are the project's *sole* safety violation. Projects that also use privileged mode, host namespaces, out-of-root mounts, or other operator-judgement features keep their instructional remediation and no button, and the blocking detail names the non-port violation first.
- The compose host-safety scan now collects all violations instead of stopping at the first, so Panelavo can distinguish a safely-fixable port-only project from one that needs manual changes.

# v0.1.52 → v0.1.53

## One-click phpMyAdmin auto-login from the Databases section

Clicking "Manage" on a database now opens phpMyAdmin already signed in as
that database's own MySQL user — no password prompt. Broker protocol bumped
to 4.

- New broker action `db-signon` (`scripts/cloudpanel-bridge.php`): after the
  usual site-write authorization and verification that the database belongs
  to the site, the root bridge reads the database user's password through
  CloudPanel's own encrypted store (`DatabaseUser::getDecryptedPassword()`)
  and writes it into a one-time token file under the database-manager site
  user's private `~/.pma-signon/` directory (0600, 60-second expiry, stale
  tokens swept). The manager site is resolved from CloudPanel's own site
  records; only the unguessable 64-hex token ever reaches the browser.
- New `scripts/pma-signon.php`, installed as `signon.php` in the manager's
  webroot: consumes the token exactly once (deleted before use), loads the
  credentials into phpMyAdmin's signon session, and redirects into the new
  signon-auth server 2 with the database preselected.
- `setup.sh` provisioning now installs `signon.php`, creates `~/.pma-signon`,
  and appends the signon server block to `config.inc.php` — all idempotent,
  so re-running upgrades existing database-manager installs. Server 1
  (cookie auth via "Open phpMyAdmin") is unchanged.
- The panel's "Manage" button POSTs `manage-login`; the live client asks the
  broker for a token and returns the signon URL, which opens in a tab created
  synchronously inside the click so popup blockers stay quiet.

Existing installs: re-run `sudo bash setup.sh` (or reinstall the root broker
and re-provision the manager) — the panel refuses to use a broker older than
protocol 4.

# 0.1.18

- Completed the CloudPanel bridge side of the Operations framework so every action and deployment plan the panel offers now actually executes.
- Detection now reports the full architecture snapshot: package.json scripts and workspaces, npm/pnpm/Yarn/Bun lockfile-based package-manager selection (ambiguous lockfiles block instead of guessing), Composer with composer.lock, Laravel, Symfony, WordPress, Django, uv/Poetry/Pipenv/pip Python manager selection, virtual environments, PM2 ecosystems, static entry points, `.env` presence, and per-tool availability.
- Framework detection covers Next.js, Nuxt, Remix, Astro, SvelteKit, Gatsby, Angular, AdonisJS, Strapi, NestJS, Express, Fastify, Koa, Hono, Laravel, Symfony, Craft CMS, WordPress, Django, Flask, and FastAPI.
- Docker Compose preflight now verifies the CLI, Compose v2 plugin, daemon reachability, and the resolved configuration, then applies a host-safety policy: loopback-only published ports, bind mounts, build contexts, secrets, and configs contained in the site root, and no privileged mode, added capabilities, host namespaces, devices, or sysctls. Compose commands run with the explicitly selected file and a stable `panelavo-<domain>` project name and are restricted to Super Admins.
- Executions re-verify every preflight precondition, take a per-site lock (`OPERATION_BUSY` when contended), run in an emptied environment whose PATH includes the site user's Node.js, Bun, Composer, and Python tool directories, and pin Poetry, Pipenv, and pip to project-local virtual environments.
- Deployment plans (Compose, Node.js, static build, PHP/Laravel, Python/Django) execute server-owned step sequences with per-step results and stop at the first failure; database migrations are never plan steps.
- Django and pip actions use the project's `.venv` interpreter; Node installs use the detected manager's deterministic command (`npm ci`, `pnpm install --frozen-lockfile`, `yarn install --immutable`/`--frozen-lockfile`, `bun install --frozen-lockfile`).
- A `FORBIDDEN` bridge result now surfaces as a 403 with a clear Super Admin message instead of a generic 502.

Rootful Compose actions previously allowed Site Managers; they now require a Super Admin, matching the documented policy. Compose projects deployed by earlier versions used Docker's default project name and should be redeployed once so containers adopt the stable `panelavo-<domain>` project name.

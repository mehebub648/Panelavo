# v0.1.53 → v0.1.54

## Large SQL imports in the phpMyAdmin database manager

Importing a real-world dump (for example a 150 MB `.sql` file) into the
database manager previously failed with Nginx's `413 Request Entity Too
Large`, because the manager site was provisioned with the Generic vhost's
default 1 MB body limit and stock PHP-FPM upload limits (2M/8M). Imports up
to 512 MiB now work.

- `setup.sh` writes a `.user.ini` into the manager's document root with
  `upload_max_filesize`/`post_max_size`/`memory_limit` of 512M and one-hour
  `max_execution_time`/`max_input_time`, and appends
  `$cfg['ExecTimeLimit'] = 3600;` to `config.inc.php` so phpMyAdmin's own
  300-second cap does not abort long imports. Both changes are idempotent
  and applied to existing installs on re-run.
- `setup.sh` patches the database-manager vhost with
  `client_max_body_size 512m` and one-hour `fastcgi_read_timeout` /
  `fastcgi_send_timeout` directives (tagged `# panelavo-db-import-limit`),
  using the same backup → `nginx -t` → restore-on-failure safety net as the
  panel vhost's proxy limits.
- `DEPLOYMENT.md` documents the new import limit.

Existing installs: re-run `sudo bash setup.sh` to apply the new limits.

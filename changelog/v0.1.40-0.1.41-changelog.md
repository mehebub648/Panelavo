# 0.1.40 → 0.1.41

- Panelavo now guarantees that each CloudPanel site user can read, write, and traverse its full application root, including files owned by containers or other runtime users.
- Existing project paths receive a site-user POSIX ACL and directories receive an inherited default ACL for future files, without changing ownership or granting world access.
- `setup.sh` now installs the required `acl` package.
- The root-owned broker protocol is now version 2. Existing installations must run the trusted `setup.sh` once before using the in-panel updater.

# 0.1.19

- Added click-to-fix remediation to Operations preflight checks: blocked checks that Panelavo can safely repair now show an explicit fix button next to the "How to fix" guidance.
- Available fixes: install Docker Engine with the CLI, Buildx, and Compose v2 plugin; install the Compose v2 plugin alone when the CLI already exists; enable and start the Docker daemon; and install Composer when a PHP site needs it.
- Installers always target the latest supported release from the official upstream source instead of a possibly outdated distribution package: Docker packages come from Docker's official APT repository (configured for the detected Debian/Ubuntu release and CPU architecture), and Composer comes from getcomposer.org with SHA-384 installer signature verification.
- Fixes are Super Admin-only, individually confirmed in the UI, executed as exact argument arrays without a shell, serialized behind a host-wide lock (`OPERATION_BUSY` when contended), and report per-step results in the operation output, stopping at the first failed step.
- Non-Super Admins see the fix with a clear "Host software changes require a Super Admin." notice instead of a runnable button.
- Automatic Docker installation is limited to Debian and Ubuntu hosts; other distributions keep the manual remediation guidance.

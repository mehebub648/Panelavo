# 0.1.36 to 0.1.37

- Install and verify the immutable CloudPanel broker before replacing legacy sudo access.
- Remove Panelavo-owned passwordless access to raw PHP and `clpctl`.
- Block self-updates before deployment when the required broker is missing, unhealthy, or incompatible.
- Use dependency-aware readiness for installer verification and keep port 10443 private.
- Add a CI regression check for the broker, sudoers, readiness, and updater contract.
- Correct the persisted installed version during self-update handoff.

Existing installations must run `sudo bash setup.sh` once from a trusted checkout before using the in-panel updater. The migration installs the root-owned broker and removes the legacy sudo rule.

# v0.1.55 to v0.1.56

- Show actionable updater validation failures directly in Panel Settings.
- Preserve the trusted `setup.sh` instruction when the root-owned broker is missing, unavailable, or incompatible.
- Keep unexpected build and deployment details in the private update log.

This release adds no new migration. An older or incompatible installed broker still requires `sudo bash setup.sh` from a trusted checkout before retrying the in-panel update.

# 0.1.35 to 0.1.36

- Persist request-throttling counters in the protected Panelavo data directory so restarts cannot reset login or mutation limits.
- Accept client addresses only from the proxy-overwritten `X-Real-IP` header and only when they are valid IP addresses.
- Add restart-persistence and spoof-resistance tests for the rate limiter.

Existing installations create `.data/rate-limits.json` automatically. No manual migration is required.
